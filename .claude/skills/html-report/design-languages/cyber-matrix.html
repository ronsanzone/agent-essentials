design language :: cyber-matrix

cyber-matrix terminal-only register

A monolithic, uncompromising design language for hacker-manifesto reports, protocol disclosures, and any document that should look like the operator never left the shell. One typeface. Black background. Phosphor green prose. Hierarchy from weight and size only — never from font family.

01

personality

cyber-matrix is a terminal emulator pretending to be a publishing system. There is one typeface — IBM Plex Mono. Hierarchy comes from weight (400/500/600/700) and size. The accent colors are matrix green for primary, cyan for secondary, and pure red for alarm. The background is pure black #000000 and the prose is matrix green #00FF41. The result reads like a terminal that decided to typeset itself.

When the document and the shell use the same vocabulary, the operator never context-switches.

Every section is wrapped in [BEGIN] / [END] markers. Headings prefix with >>>. Subheadings prefix with >>. Inline labels prefix with #. The slash-slash // is reserved for inline annotations and quoted material. The arrow chain is the brand's syntactic spine — replace it with chevrons or arrows and the language stops feeling like itself.

02

when to deploy

deploy when

Protocol disclosures, exploit writeups, hacker-manifesto-style essays, terminal-tool documentation where the tool itself is a CLI. Anywhere the audience already lives in a terminal and would resent a more decorated register.

do not deploy when

Audience expects prose readability over visual identity. Long sequential reading where a serif/sans split would reduce fatigue — pick editorial-parchment. Anywhere the content is celebratory, illustrative, or sales-adjacent.

voice rules

  • One typeface. No exceptions. Even pull quotes are mono.
  • Sentence-case for headings (heading prefix >>> does the work that ALL CAPS would do elsewhere).
  • Section blocks wrap with [BEGIN] / [END] markers above and below.
  • Cards wrap with [BEGIN] / [END] labels inset into the border, like a fenced terminal output region.
  • Pre blocks carry a $ prompt glyph in the left margin — they look like shell history, not snippets.
  • No italic. No serif. No alternate font. The whole document is one face.

05

anti-patterns

  • No serif. No proportional sans. No second typeface for any role.
  • No italic. Use --text-dim for de-emphasis instead.
  • No rounded corners. Borders are sharp 0px.
  • No gradients. No drop shadows except phosphor glow on the progress bar and blinking cursor.
  • No light theme. The black is load-bearing.
  • No emoji ever — substitute ASCII glyphs (>, $, #, !!!, [OK]).
  • No additional accent colors. No additional brand-warm tones.
  • No photographic imagery. SVG only.
  • No ALL CAPS for body prose — the prefix glyphs (>>>, #, //) carry the visual signal already.

07

diagram philosophy

Diagrams are literal ASCII — boxes drawn from Unicode box-drawing characters in SVG <text> blocks, not <rect> shapes. The vocabulary is typographic: ┌─┐ │ └─┘ for default, ╔═╗ ║ ╚═╝ for emphasis, ──> and ═══> for arrows. Role shows through ink color and weight, never fill. The aesthetic is "terminal output piped into the page."

SVG typography

Every diagram is a single <text> block with xml:space="preserve" and stacked <tspan x="10" dy="1.2em"> lines. Color and weight come from nested <tspan class="..."> spans inside each line. No <rect> ever.

.ascii      { font: 500 14px "IBM Plex Mono", monospace; fill: var(--text); white-space: pre; }
.ascii .dim  { fill: var(--text-dim); }
.ascii .hot  { font-weight: 700; }
.ascii .ext  { fill: var(--cyan); }
.ascii .step { fill: var(--cyan); letter-spacing: 0.06em; }
.ascii .red  { fill: var(--red); font-weight: 700; letter-spacing: 0.12em; }
.ascii .note { fill: var(--text-dim); }
LOCKEDBox-drawing glyphs only — ┌─┐ │ └─┘ for default, ╔═╗ ║ ╚═╝ for hot & alert regions. Arrows are ──> (default) and ═══> (heavy, from hot). Step counters as [NN] in cyan. Notes prefixed // in dim green.
FREELifeline column widths, presence of step numbers on simple diagrams, exact line spacing (1.15–1.30em).

alert region pattern

When a sequence has a failure leg, wrap the affected lines in a literal ╔════ ALERT ════╗ … ╚═════════╝ frame in red. The lines inside keep their dim-green lifelines but the frame and inline labels (503 UNAVAILABLE, RETRY_POLICY_ENGAGED) render in saturated #FF0000. A leading ▮ block-glyph stands in for the old red tick rectangle. Red is the loudest thing in the language; when it appears it lands hard.

LOCKEDRegion opens with ╔════ ALERT ════╗, closes with ╚════════════╝. All ALERT-frame glyphs render in var(--red) bold. The ▮ glyph precedes the snake_case retry label.
FREEWidth of the ALERT frame, exact label copy, presence of the inline backoff … annotation.

arrow vocabulary

Single-line ──> for default transitions, double-line ═══> for the focused/hot transition. Return paths shorten to <─ ─ (dashed with spaces) in dim green. Async fan-outs use the same ─ ─ ─ spaced form. Warn arrows live entirely inside the ALERT frame and inherit its red.

caption

Mono, var(--text-dim), 12px, centered, prefix //. Caption describes the ASCII rendering — the glyphs themselves are part of the content.
