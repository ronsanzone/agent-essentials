Design Language · Editorial Register

Stripe Blog, set in serif long-form

An editorial register: Stripe's color palette laid over a literary type system. For essays, reflections, retrospectives, and any long-form writing that wants the Stripe trust signal but the breathing room of a serif book page.

Long-form Reading: ≈ 25 min
LockedSource Serif Pro 700 h1 at clamp(40px,5vw,64px), italic purple <em>, serif italic lede at 22px, 640px text column max-width, purple as single accent (no blue, no secondary).
FreePadding, hero meta as .meta-row key/values or .meta pill chips, exact lede line-length.
01

Personality

Stripe Blog is the editorial register turned up — what Stripe's blog would look like if it leaned harder into long-form essay aesthetics. Source Serif Pro for body and display. Inter only for small functional UI (eyebrows, h4 labels, meta keys). IBM Plex Mono for code. The recognizable Stripe purple #635BFF as the single accent — no blue, no secondary, no warm balance.

Stripe’s color palette laid over a literary type system: the trust signal of fintech, the reading comfort of a book page.

The 640px text column is the defining geometric constraint. Body lines are deliberately narrow so prose reads at book pace rather than screen-scrolling pace. Wider components (tables, diagrams, callouts) break out to 760px — wide enough to breathe, narrow enough to stay in the editorial register. The result feels nothing like the press page; it feels like a magazine essay.

This is the variation a long-form retrospective on the Stripe blog would use if Stripe leaned harder into editorial. Most Stripe blog posts in production sit between this and stripe-press — this language is the editorial extreme.

02

When to pick this

Pick this when

Engineering essays. Retrospectives. Year-in-review reports. Reflections on architecture or process. Anything that reads like prose first and reference second. Where the reader will sit and read for 20+ minutes without scrolling reference.

Don't pick this when

Reference docs — use stripe-docs. Product announcements — use stripe-press. Reports that are dense with code and tables; the narrow 640px column wastes width for those.

03

Locked brand stack

The serif/sans split is load-bearing. Source Serif Pro does the reading work — body, display, ledes. Inter does the small functional work — eyebrows, h4 labels, meta keys, anything that needs sans-typed UI clarity. IBM Plex Mono handles code. Söhne, again, is the Stripe brand sans; Inter is the Google-Fonts fallback.

Voice rules

  • H1 carries an italic serif em phrase in purple — the canonical editorial title-with-subhead pattern.
  • The lede is always serif italic. This is the strongest editorial signal the language carries.
  • Body uses italic for emphasis, never bold. Bold is reserved for the occasional in-line term gloss.
  • Mono caps with wide letter-spacing for eyebrows (0.20em) and meta keys (0.15em).
  • Pull quotes are large (≥28px), serif italic, in purple, with no decorative borders or quotation marks — let the type carry the weight.
  • The 640px text column is non-negotiable for prose. Tables, diagrams, callouts may break to 760px max.
  • Single accent: purple only. No link-blue, no navy as a secondary, no amber outside the warn slot.

Diagram-kit aliases

Both --accent and --secondary point at purple. The role distinction in diagrams shows through tone (purple vs purple-soft) rather than hue.

--accent:         var(--purple);
--accent-soft:    var(--purple-soft);
--secondary:      var(--purple);
--secondary-soft: var(--purple-soft);
Warning: Resist the urge to introduce a second accent for variety. Single-accent discipline is what gives this register its editorial purity. If you find yourself reaching for navy or link-blue as a secondary, switch to stripe-press instead.
05

Anti-patterns

  • No sans body. Source Serif Pro is the body face. Inter is small-UI-only.
  • No press-style display headings (h1 at 88px is wrong here — 64px is the ceiling).
  • No wide text columns. 640px max for prose; 760px max for break-out components.
  • No second accent color. Purple only. (See §04.)
  • No drop-shadows heavier than 0 1px 3px rgba(10, 37, 64, 0.03). The register is flat.
  • No warm-paper backgrounds. The surfaces are cool — same as the other Stripe registers.
  • No bold for body emphasis. Italic carries emphasis here.
  • No decorative quotation marks on pull quotes. The italic serif does the work.
  • No emoji.
  • No method-typed pills, no GET/POST color-coding — that's the docs register.

07

Diagram philosophy

Diagrams in editorial writing illustrate an argument, not a system. The Blog idiom borrows from the explanatory graphics in long-form newspaper features: 1px purple outlines, white-fill boxes, labels keyed to a paragraph that explains each panel in prose. The accent is delicate (#635BFF hairline strokes), the typography is the body serif, and the alert region is a thin dashed purple frame — not a filled tint.

SVG typography

Everything in the diagram is Source Serif Pro. There is no monospace step register and no display face — the diagram is part of the prose:

.actor     { font: 600 13px "Source Serif Pro", serif; fill: var(--navy-deep); }
.msg       { font: italic 12px "Source Serif Pro", serif; fill: var(--text); }
.note      { font: italic 11px "Source Serif Pro", serif; fill: var(--text-light); }
.sub-lbl   { font: 600 12px "Source Serif Pro", serif; fill: var(--purple); }
.alert-txt { font: italic 12px "Source Serif Pro", serif; fill: var(--warn); }
LockedSource Serif Pro throughout — no mono, no caps. Labels OUTSIDE boxes. Sub-letter keys (a)(b)(c) in purple, keyed to a legend paragraph immediately below the figure.
FreeNumber of panels, whether to include a legend below or run the explanation inline in body prose. Roman vs alphabetic keys are both acceptable; alphabetic is the default.

Alert region pattern

Failure regions are a thin dashed purple-family frame — never a filled tint. The frame uses var(--warn) stroke at 1px with stroke-dasharray: 3 3, no fill. An italic small caption inside the frame names what is going wrong; an inner nested rectangle (also outline-only) carries the dominant alarm message.

LockedFrame: 1px --warn dashed, no fill. Inner pill: 0.75px --warn outline, no fill. Alert text in Source Serif italic — same face, same italic register as the inline notes.
FreeFrame dimensions, exact wording, presence of the inner pill (small alert regions may omit it).

Arrows

Default: var(--purple) hairline at 1px — delicate strokes feel editorial. Muted return paths: --text-ghost. Async: stroke-dasharray: 5 3. Arrowheads are small (7×7) and matched to the line color.

Caption convention

Captions begin with an italic Figure N. prefix in --text, followed by a single sentence that names what the figure shows. The full explanation lives in the legend paragraph immediately beneath — a <ul class="panel-legend"> whose entries open with the matching (a)(b) key. The legend is part of the figure, not the surrounding prose.
