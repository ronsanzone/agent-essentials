Design Language

Nord Snow, arctic palette in daylight

A cool, Snow-Storm-bright register that proves Nord's palette logic survives light mode — Scandinavian-furniture-catalog calm for public docs and reader-friendly references.

Status: candidate Mode: light v1
LockedIBM Plex Sans 300 h1 with deep-blue italic <em>, IBM Plex Sans 300 lede, IBM Plex Mono caps eyebrow with --nord14 green dot, nord5 → nord6 linear gradient.
FreePadding, lede max-width, hero footer as .meta-row key/values or .meta pill chips.
01

Personality

Nord Snow is Nord in daylight. Snow Storm — the three brightest values in the palette — supplies surfaces, while Polar Night supplies ink. The Frost layer steps forward as the lead voice: --nord10 deep blue carries links, italic-em, active TOC, and primary diagram nodes. It has more authority on a light background than --nord8's bright teal would — bright teal evaporates against Snow Storm surfaces; deep blue holds.

The secondary is the gentle Aurora green --nord14 — eyebrow dot, marginalia counter, pill outlines. Green is the one Aurora color that doesn't read as a warning or a celebration; it reads as a calm parenthetical. Warn stays on --nord11 Aurora red.

Two reports in this language should feel like two pages of a well-set Scandinavian furniture catalog — visibly the same voice, visibly not xeroxed.

The body is IBM Plex Sans because Plex's slightly more humanist forms compensate for the loss of warmth — it keeps the document approachable where Inter would feel clinical. Plex Mono carries code and labels with the same humanist softening.

02

When to pick this

Pick this when

Public-facing developer docs, library landing pages, marketing-adjacent technical material. Reports for readers uncomfortable with dark themes. References that will be printed or pasted into ticketing tools where light surfaces dominate. Anywhere "Nord, but not a code editor" feels right.

Don't pick this when

The audience expects warmth (use editorial-parchment). The report needs heavy multi-status color signaling (use nord-aurora). The reader lives in a terminal and prefers dark-canvas documents (use nord-polar).

Voice rules

  • Italic phrases inside h1 use <em> and are colored --nord10 deep blue.
  • Body italics are permitted; never bold body prose for emphasis — use italic.
  • Mono caps with 0.12–0.18em letter-spacing for eyebrows and meta keys.
  • Pull quotes are large (~25px), Plex Sans light, deep-blue, set off by a thin left rule.
  • The eyebrow dot is always --nord14 Aurora green — the language's smallest secondary touch.
  • Generous whitespace — section gaps and line-heights are slightly looser than the dark variants. The Scandinavian-catalog feeling lives in the breathing room.

Warning: Do not promote --nord8 bright teal to the lead-accent slot. On light Snow Storm surfaces, bright teal washes out — it lacks the contrast to anchor a paragraph. Deep blue --nord10 is the deliberate choice; demoting it breaks the language.
05

Anti-patterns

  • No warm tints. This is COOL light — not parchment, not cream, not warm grey. If the background hints toward beige or ivory, the language is broken. Compare against editorial-parchment: that file is warm light; this file is cool light. They must not be mistaken for each other.
  • No warm accents anywhere — no terracotta, no amber, no rust. Warn stays Aurora red --nord11, which is desaturated and cool-leaning.
  • No serif display fonts — the language is sans-only.
  • No system-font defaults (no Arial, no Roboto, no -apple-system as head of stack).
  • No bright-teal --nord8 as a lead accent on these surfaces (see palette warning above).
  • No dark backgrounds in this file — that is nord-polar.
  • No simultaneous use of multiple Aurora colors as semantic signals — that is nord-aurora.
  • No heavy shadows. Drop-shadow ceiling is 0 6px 20px rgba(46, 52, 64, 0.04).
  • No emoji in headings or callouts.

07

Diagram philosophy

Diagrams here are quiet and bright — clean Snow Storm surface, thin cool-grey borders, deep-blue arrows. Text inside diagrams uses IBM Plex Sans (matching body) to keep diagram and prose in conversation. Aurora colors stay out of the diagram surface except for warn (--nord11); Aurora green leads external nodes as the secondary accent.

LockedInline annotations use Plex Sans italic in --ink-faint. Step numbers and alert text use Plex Mono with letter-spacing — darkened Aurora green for steps, --nord11 for alerts.
FreeWhether to use step numbers at all (small diagrams omit them), exact letter-spacing (0.10–0.16em), annotation placement.

Alert region pattern

When a diagram has a failure path, wrap the affected messages in an Aurora-red-tinted dashed rectangle and place an inline alert pill (nord11 @ 0.12α fill + --nord11 border + --nord11 mono caps text). The pattern is visible in the sequence diagram above between messages 03 and 04.

LockedWarn-tinted background at rgba(191, 97, 106, 0.05) + dashed --nord11 border. Inline alert pill stacks a colored dot, a mono-caps label, and a Plex italic detail.
FreeRegion size, exact alert pill copy, whether to include the colored dot.

Arrow color

Default arrows: --nord10 deep blue. 2px stroke for primary flow, 1.5px for muted/return. Failure-path arrows use --nord11; async dispatch is dashed deep blue.

Caption

Plex Sans light italic, --ink-faint, 13px, centered under the diagram with margin-top: 14px.

08

Exemplar reports

Read at least one before writing a fresh report. Exemplars are evidence of the language deployed in real content — not templates to clone.

  • nord-snow/library-landing-overview.html — public library landing-page report; the language at full scale.
  • nord-snow/api-reference-summary.html — public API reference summary at smaller scale; useful when the report is mid-length.

The two exemplars are visibly different in composition but instantly recognizable as the same language. That's the goal: locked brand stack, free composition.

If two Nord Snow reports feel xeroxed, one is wrong. If two feel like different brands, both are wrong.
