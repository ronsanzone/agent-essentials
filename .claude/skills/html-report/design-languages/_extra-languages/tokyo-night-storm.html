Design Language

Tokyo Night Storm, night-shift register

A dark cityscape for engineers reading at 11pm — electric indigo lit by blue and violet neon, geometric Inter at heavy weights, no nostalgia, no warmth.

Status: candidate Audience: engineer-internal v1
LockedInter 800 h1 with letter-spacing -0.035em, blue italic <em> at weight 400, mono caps eyebrow with 6px purple dot (with neon glow), bg-sidebar → bg-editor vertical gradient.
FreePadding, lede max-width, hero footer as .meta-row key/values or .meta pill chips.
01

Personality

Storm is the canonical Tokyo Night — that specific deep indigo-navy where most dark themes go cold and grey. The base background is #24283b: it has blue in it. Look at it next to GitHub Dark or Dracula and the difference reads immediately — Tokyo Night is a night sky, not a charcoal slate.

Typography is geometric and confident. Inter at 800 for display, Inter at 400 for body and italic accents. No serif anywhere. The visual rhythm comes from weight contrast (300 / 400 / 600 / 800) and a tight negative letter-spacing on display. JetBrains Mono carries eyebrows, code, marginalia, and the small caps moments. Blue is the primary accent — it's where Tokyo Night gets its identity from VS Code function colors. Purple is secondary — keywords, pill chrome, the eyebrow dot. Orange is reserved for warn.

Storm should feel like a senior engineer's IDE at night — backlit, precise, no decoration that didn't earn its pixels.

The blue/purple duo is load-bearing. Blue belongs to active state, links, hot diagram nodes, and the italic <em> phrases inside titles. Purple belongs to numbers, the eyebrow dot, pill borders, and section .num markers — wherever a small accent must register without competing with the prose. Cyan exists for hover and code types only.

02

When to pick this

Pick this when

Late-shift operator runbooks, on-call playbooks, deep technical tours read on a dark monitor. Reports about CLI tooling, terminal output, or developer-internal infrastructure where the reader's eyes have already adjusted to a dark editor. Code-heavy documents that benefit from token colors that match a working IDE.

Don't pick this when

The report will be projected in a sunlit room, exported to PDF for print, or sent to executive stakeholders who read on bright displays. Outcomes-forward postmortems that need a register of formality — use a light language. Documents under ~5 minutes — Storm's chrome overpowers short content.

Weight ladder

  • h1.title: weight 800, letter-spacing: -0.035em — the heaviest moment in the document.
  • h2: weight 700, letter-spacing: -0.025em.
  • h3: weight 600, letter-spacing: -0.015em.
  • Body: weight 400. Italic em in titles: weight 400 italic.
  • Pull quote: weight 300 italic — the only light-weight moment in the language.

Voice rules

  • Italic Inter for ledes, pull quotes, and the italic phrase inside h1: <h1>Title, <em>italic phrase</em></h1>.
  • Mono caps with letter-spacing 0.18em for eyebrows; 0.12em for meta-row keys and section numbers.
  • Never bold body prose for emphasis — use italic instead. Bold is reserved for callout labels.
  • The eyebrow dot is always var(--purple) with a soft glow shadow — it is the brand's smallest neon touch.
  • Headings are always Inter. No serif fallback. No display fonts.

05

Anti-patterns

  • No serif fonts. Storm is Inter-only for prose, JetBrains Mono for technical. Any Fraunces, Newsreader, or Georgia breaks the geometric register.
  • No warm background tints — no parchment, no cream. The base must be the exact indigo #24283b.
  • No pure white (#fff) for ink. Use --fg-primary (#c0caf5) — Tokyo Night's lavender-tinted off-white.
  • No pure black backgrounds. The whole point is indigo, not OLED-grade darkness.
  • No heavy drop shadows. Cards lift with 0 8px 24px rgba(15, 17, 28, 0.35) and a 1px inner highlight — anything heavier reads as 2014 material design.
  • No bright neon green or hot pink as primary accents. Tokyo Night's saturation is deliberately restrained — colors sit at ~60–70% saturation, never 100%.
  • No emoji in headings or callouts. The mono-caps eyebrow and the purple dot carry the affordance.
  • No glow shadows on text. The only glow in the language is the 8px purple shadow on the eyebrow dot.

07

Diagram philosophy

Diagrams in Storm are night-shift schematics — boxes with thin indigo borders, generous padding, bg-active fill, blue arrows. Text inside diagrams uses Inter (matching body) to keep the diagram and prose in conversation. Hot nodes get the blue accent wash that mirrors function-name color in a Tokyo Night IDE.

SVG typography

Base text classes:

.txt   { font: 500 14px "Inter", sans-serif; fill: var(--fg-primary); }
.small { font: 400 12px "Inter", sans-serif; fill: var(--fg-secondary); }

Language extensions — used when a diagram needs inline annotation, step numbering, or alert text (see the annotated sequence above):

.note      { font: italic 400 12px "Inter", sans-serif; fill: var(--fg-faint); }
.step      { font: 500 11px "JetBrains Mono", monospace; fill: var(--purple); letter-spacing: 0.12em; }
.alert-txt { font: 600 11px "JetBrains Mono", monospace; fill: var(--orange); letter-spacing: 0.14em; }
LockedInline annotations use Inter italic in fg-faint. Step numbers and alert text use JetBrains Mono with letter-spacing — purple for steps, orange for alerts.
FreeWhether to use step numbers (small diagrams omit them), exact letter-spacing (0.10–0.16em), annotation placement.

Alert region pattern

When a diagram has a failure path or exceptional region, wrap the affected messages in an orange-tinted dashed rectangle and place an inline alert pill (orange-soft fill + orange border + orange-colored mono caps text). The pattern is visible in the sequence diagram above between messages 03 and 04.

LockedWarn-tinted background uses rgba(224, 175, 104, 0.08) + dashed orange border. The inline alert pill stacks an orange dot, a mono-caps label, and an italic Inter detail.
FreeRegion size, exact alert pill copy, whether to include the colored dot.

Arrow color

Default arrows: var(--blue). 2px stroke for primary flow, 1px for secondary references. Return paths in a sequence diagram use arrow-muted with var(--border) color (#3b4261) — direction is legible at a glance because returns are dimmer.

Caption

Italic Inter, var(--fg-faint), 14px, centered under the diagram with margin-top: 12px.

Exemplar reports

Read at least one before writing a fresh report in Storm. Exemplars are evidence of the language in real content — not templates to clone. This styleguide is the canonical reference for vocabulary; exemplars show vocabulary deployed.

  • None shipped yet — Storm is a candidate language. First exemplar will be commissioned once the language is promoted from _candidates/ into design-languages/.
  • Candidate first-use: an on-call runbook or terminal-tooling ramp-up where the dark register is justified by the subject matter.

Two Storm reports should be visibly the same language but visibly not xeroxed — locked brand stack, free composition.

If a Storm report doesn't make you want to dim the monitor lights, it isn't Storm yet.
