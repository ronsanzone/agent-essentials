Design Language · Series C

Cyber-Corpo, boardroom register

A measured cyberpunk dossier register — for product surfaces, technical white papers, and corporate-cyber documentation that needs the edge without the glitch. The variation a megacorp's polished UI team would actually ship.

Series C · v1 Audience: Product Confidential
LOCKEDBarlow Condensed 600 uppercase h1, cyan light-weight 300 em, Inter 300 lede, mono cyan eyebrow with cyan square dot + glow, dark navy background with subtle cyan inset highlight on top edge.
FREEPadding, lede max-width, meta as pills or key/value rows, glow intensity.
01

Personality

Cyber-Corpo is the version of cyberpunk that survives a procurement review. The aesthetic is dark navy #0a0e17, cyan as a measured lead accent (never as a billboard), Barlow Condensed for the display register so headings carry editorial weight without shouting, and Inter for body so sustained reading isn't punishing. The mono is IBM Plex — corporate-credible, terminal-aware. The result reads as a luxury dossier with a cyber edge — boardroom, not back-alley.

Two reports in this language should feel like two chapters of the same megacorp annual — different content, identical institutional voice.

Cyan is the lead, but cyan is restrained. It carries links, the title em phrase, the active TOC item, primary diagram nodes, table headers, and inline code. Magenta is reserved for success/confirmation (used sparingly — this register doesn't celebrate often). Red is reserved for failure. The trio is calibrated for a product surface where the wrong tone reads as glitch when the room expects commerce.

02

When to deploy

DEPLOY WHEN

Product overviews, technical white papers, enterprise security dossiers, internal strategy documents with a cyber-product angle. Anywhere the audience expects polish and the content has technical or security DNA.

DO NOT DEPLOY WHEN

The audience is fully internal-engineering and a more confrontational register is fine — pick cyber-2077. The document is long-form sequential prose where condensed display tires the eye — pick editorial-parchment.

Voice rules

  • Display headings (h1/h2/h3) and pull quotes use Barlow Condensed uppercase — never sentence-case.
  • Body prose is Inter 400, sentence-case. The em inside h1 uses Inter Condensed 300 in cyan — light weight is the contrast against the heavy h1.
  • Eyebrow, captions, pills, and contract labels all use IBM Plex Mono with letter-spacing 0.10–0.24em.
  • No italics. No serifs. Hierarchy comes from face + weight + size + caps.
  • Subtle inset 1px highlight on the top edge of cards — the only "lighting" in the design.

05

Anti-patterns

  • No yellow primary. Yellow reads as cyber-2077, not cyber-corpo.
  • No serif typefaces. No italic for emphasis — use weight or color.
  • No purple/violet anywhere. This is not synthwave.
  • No rounded corners over 4px. Buttons, cards, pills are crisp.
  • No drop shadows beyond the locked 1px inset highlight and the subtle below-shadow on cards.
  • No light theme. This is dark-only.
  • No emoji. No mascots. No illustration with personality.
  • No bright photography. Treat any imagery as SVG schematic or product render.
  • No more than one magenta element per visible viewport — magenta is rare on purpose.
  • No paper grain or textile texture.

07

Diagram philosophy

Diagrams are restrained schematics. 4px corner radius on nodes, 1px borders, navy fills consistent with cards. Text inside diagrams uses Inter (matching body); coordinates and step markers use IBM Plex Mono. The cyan lead carries the active path; magenta and red appear only where the role demands.

FREEWhether to include step coords on small diagrams, tracking (0.12–0.20em on steps), Inter weight inside boxes (400–600).

Alert region pattern

When a sequence has a failure leg, wrap the affected messages in a red-tinted dashed rectangle. The inline alert callout uses cyan — not red — so the recovery action reads as primary signal while the failure stays as context. This is the corpo register: red marks the problem, cyan marks the response. The alert pill is the loudest element on the page after the failure arrow itself.

LOCKEDRegion: rgba(220,38,38,0.08) + dashed red border. Failure message in mono red. Recovery pill in 10% cyan fill + 1.5px cyan border, with cyan tick block, mono uppercase cyan label, neutral Inter detail.
FREERegion size, pill copy, whether the recovery pill includes a tick block or just bare text.

Arrow color

Default arrows: var(--cyan), 1.8px stroke. Async arrows: cyan dashed (6 3). Return paths use var(--border-hi) muted. Warn arrows: var(--red), 2.2px stroke — failure leg always heavier than the success leg.

Caption

IBM Plex Mono, var(--text-dim), 12px, centered under the diagram with margin-top: 14px. See the captions above.

Exemplar reports

Read at least one before deploying. Exemplars are evidence of the language fielded — not templates to clone. This file is the canonical vocabulary; exemplars show vocabulary deployed.

  • ~/notes/04_Research/product-runtime-overview.html — full product surface, every component, dense sequence diagrams. Reference for "everything this language can do at full scale."
  • ~/notes/04_Research/enterprise-security-dossier.html — shorter security white paper; same language at mid-scale.

Two reports in this language should feel like two chapters of the same megacorp annual: visibly the same institutional voice, visibly different content. If they feel xeroxed, the author leaned too hard on locked elements. If they feel like different brands, someone reached for unauthorized accents.

Cyan leads. Magenta accents. Red alarms. Anything else dilutes the signal.
