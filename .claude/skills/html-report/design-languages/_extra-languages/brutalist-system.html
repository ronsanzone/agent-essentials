Design Language

Brutalist System, unstyled on purpose

The default browser, maximally. No fonts loaded. No rounded corners. Links are #0000EE. The absence of design decisions IS the decision.

Status: shipped Audience: technical-manifesto v1
LockedSystem sans h1, 1px solid black borders, native browser link blue, no border-radius, no shadows, no gradients. Hero has a bottom rule and nothing else.
FreePadding, title case (upper vs sentence), pill content, hero footer as meta-row or pills.
01

Personality

Brutalist System is anti-design as design philosophy. The medium is the statement. It is the right choice for technical manifestos, personal developer sites, and tools-for-tools documentation where ornamentation would undermine credibility. The aesthetic principle: consistency in restraint.

If a reader notices the design, the design has failed. If a reader notices the absence of design, that is the design working.

Body is the operating-system sans stack. Code is "Courier New". No Google Fonts are loaded for body or display — the only optional CDN dependency is Courier Prime for code blocks if a renderer wants a slight upgrade. Backgrounds are #FFFFFF and #F5F5F5. Text is #000000. Links are #0000EE — the native browser hyperlink color, untouched. Visited links are #551A8B. The brand identity is "we didn't bother styling them."

Red (#FF0000) is the only secondary accent and appears sparingly: marginalia counters, the eyebrow dot, alert states. Everything else is black on white.

02

When to pick this

Pick this when

Technical manifestos. Standards documents. Personal developer sites that signal seriousness through restraint. Documentation for tools-for-tools where ornament would feel out of place. RFC-style writing.

Don't pick this when

The audience expects polish (external sales, marketing). The content is narrative or persuasive — restraint reads as indifference there. Reports that must reassure rather than instruct.

Font stack variables

--sans:  system-ui, -apple-system, "Helvetica Neue", sans-serif;

Font role assignments

RoleFontNotes
Body prosevar(--sans)16px, line-height 1.5
Display / headingsvar(--sans) bold uppercaseHierarchy by size and weight only
Code, eyebrows, marginalia, pills, meta keysvar(--mono)Courier New, no substitute

Voice rules

  • Headings are uppercase. Letter-spacing is unmodified or near-zero.
  • Bold for emphasis. Italic is allowed but unstyled.
  • No display-only typography. The same sans renders body and heading.
  • Inline code is bordered (1px solid black) — code stands out by geometry, not color.
  • Native browser blue (#0000EE) for links. Visited purple is allowed. Hover state is a background wash, not a color change.

05

Anti-patterns

  • No border-radius. Ever. border-radius: 0 is the default and the rule.
  • No box-shadow. Shadows imply elevation; this language is flat.
  • No gradients. Solid fills only.
  • No styled focus rings. The browser default is correct.
  • No webfonts for body or display. System stack only.
  • No emoji in headings, callouts, or marginalia.
  • No "tasteful" off-blacks. Text is #000000, not #222.
  • No "tasteful" off-whites. Background is #FFFFFF, not #FAFAFA.
  • No animation beyond the progress-bar width transition (locked at 0.05s linear).

07

Diagram philosophy

Diagrams are 1px black lines on white. Boxes are rectangles. Arrows are black. The hot role is a grey-filled rectangle — there is no separate "accent" color. Failure paths and the warn role use #FF0000 stroke and that is the only place red appears in diagrams.

Role-to-palette map

RoleFillStrokeNotes
default#FFFFFF#0000001.5px stroke, no rx (square corners).
hot#F5F5F5#0000002px stroke for emphasis.
store#FFFFFF#0000002px stroke, doubled border via inner rect.
external#FFFFFF#000000Dashed stroke (stroke-dasharray: 6 3).
warn#FFFFFF#FF0000The only red. 2px stroke.
future#FFFFFF#888888Dashed, ghosted stroke.

SVG typography

.txt   { font: 14px system-ui, sans-serif; fill: #000000; }
.small { font: 12px system-ui, sans-serif; fill: #555555; }
.note  { font: 12px "Courier New", monospace; fill: #555555; }
.step  { font: 700 11px "Courier New"; fill: #FF0000; }
.alert-txt { font: 700 11px "Courier New"; fill: #FF0000; }
LockedStep numbers and alert text are red Courier New uppercase. Step labels use [NN] bracket format. The only red in a diagram is failure-path / alert.
FreeWhether to include step numbers (small diagrams omit them), exact letter-spacing (0.06–0.12em).

Alert region pattern

Wrap failure-region messages in a grey-filled rectangle with a 1px dashed red border. Place an inline alert pill (white fill, red border, red Courier text) explaining the consequence. Visible in Fig. 02 between steps 03 and 04.

LockedGrey fill (#F5F5F5) + 1px dashed red (#FF0000) border. Inline alert pill: white fill, solid red border, red Courier caps text, optional 8px red square dot at left edge (no circles — squares only).
FreeRegion size, exact alert-pill copy, presence of the leading red square dot.

Arrow color

Default arrows: black. Muted return paths: #555555. Warn / failure-path: red #FF0000. Async: black with stroke-dasharray: 6 3. Never colored otherwise.

Caption

Mono uppercase Courier New, #000000, 12px, left-aligned under the diagram. Captions start with a Fig. NN prefix.
