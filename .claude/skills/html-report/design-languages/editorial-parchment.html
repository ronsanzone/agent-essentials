Design Language

Editorial Parchment, set in long form

A typographic register for ramp-up tours, operator references, and PR reviews — for engineers who will sit and read sequentially for twenty minutes.

Status: shipped Audience: engineer-internal v1
LockedFraunces h1 with opsz 144, teal italic <em>, Fraunces italic lede at opsz 32, mono caps eyebrow with 6px terracotta dot, bg-soft → bg linear gradient.
FreePadding, lede max-width, hero footer as .meta-row key/values or .meta pill chips.
01

Personality

Editorial Parchment feels like a long-form New York Review of Books essay set with a tasteful technical accent. Warm parchment paper, distinctive Fraunces serif display, clean Geist sans body, characterful JetBrains Mono for code and marginalia. Teal as primary accent — links, italic-em in titles, active TOC, primary diagram nodes. Terracotta as secondary — eyebrow dot, marginalia number, ticket pills.

Two reports in this language should feel like two articles in the same publication: visibly the same voice, visibly not xeroxed.

The serif/sans split is load-bearing. Headings and ledes are Fraunces because Fraunces is what gives the document its register. Body is Geist because long sequential reading wants a sans with a steady rhythm. Code is JetBrains Mono because the marginalia and eyebrows lean on its narrow uppercase forms. Swap any of these out and the language stops feeling like itself.

02

When to pick this

Pick this when

Dense ramp-up tours engineers will read for 20+ minutes. Operator reference docs re-read across incidents. PR review tours with file:line citation density. Technical documents read sequentially.

Don't pick this when

The report is outcomes-forward and feels more like a field report or postmortem — use a future field-report language. The report is short (under ~5 minutes reading time) and would feel overdressed in this register.

Optical-size settings

  • h1: font-variation-settings: "opsz" 144 — maximum optical size, most refined letterforms.
  • h2: font-variation-settings: "opsz" 60 (renderer may tune ±10).
  • Ledes and pull quotes: font-variation-settings: "opsz" 32.

Voice rules

  • Italic serif for ledes, pull quotes, and italic-em phrases inside h1 titles: <h1>Title, <em>italic phrase</em></h1>.
  • Mono caps with letter-spacing for eyebrows and meta-row labels.
  • Never bold body prose for emphasis — use italic instead.
  • Pull quotes are large (≥24px), serif, italicized, set off by a thin left rule.
  • The eyebrow dot is always var(--terracotta) — it is the brand's smallest terracotta touch.

05

Anti-patterns

  • No Inter, Roboto, Arial, or system-font defaults.
  • No purple gradients on white backgrounds.
  • No dark themes — this language is light-only.
  • No shadows heavier than 0 10px 30px rgba(60, 45, 20, 0.04). Heavier shadows kill the paper feel.
  • No full-bleed photographs. This is a typographic language.
  • No emoji in headings or callouts.
  • No bright/saturated accents outside the warm palette (no electric blue, no neon green).
  • No decorative gradients or patterns beyond the subtle paper grain (radial-gradient at very low opacity).

5. Pull quote .quote

The stickiest sentence in a section — the one a skimming reader must absorb. One per section maximum.

6. Details block details

Step 1. Initial step (open by default)

The first item in a series commonly carries open so context is immediately visible. Summary uses Fraunces 22px teal — it feels like a collapsible sub-heading.

Step 2. Subsequent step

Subsequent items collapse so the reader can skip well-understood steps. The terracotta + / − marker is part of this language's expression.

LockedSerif summary in teal at ~22px, rule top-border on each details.

07

Diagram philosophy

Diagrams here are quiet and precise — not decorative. Boxes have thin borders, generous padding, parchment fill, teal arrows. Text inside diagrams uses Geist sans (matching body) to keep the diagram and the surrounding prose in conversation.

SVG typography

Base text classes from diagram-kit:

.small { font: 12px "Geist", sans-serif; fill: var(--ink-soft); }

Language extensions — used when a diagram needs inline annotation, step numbering, or alert text (see the annotated sequence above):

.note      { font: italic 12px "Fraunces", serif; fill: var(--ink-faint); }
.step      { font: 500 11px "JetBrains Mono", monospace; fill: var(--terracotta); letter-spacing: 0.12em; }
.alert-txt { font: 600 11px "JetBrains Mono", monospace; fill: var(--warn);       letter-spacing: 0.14em; }
LockedInline annotations use Fraunces italic in ink-faint. Step numbers and alert text use JetBrains Mono with letter-spacing — terracotta for steps, warn for alerts.
FreeWhether to use step numbers at all (small diagrams omit them), exact letter-spacing (0.10–0.16em), annotation placement (above arrow vs. below).

Alert region pattern

When a diagram has a failure path or exceptional region, wrap the affected messages in a warn-tinted dashed rectangle and place an inline alert pill (warn-soft fill + warn border + warn-colored mono caps text) explaining the consequence. The pattern is visible in the sequence diagram above between messages 03 and 04.

LockedWarn-tinted background uses rgba(154, 99, 0, 0.06) + dashed warn border. The inline alert pill stacks a colored dot, a mono-caps label, and a Fraunces italic detail.
FreeRegion size, exact alert pill copy, whether to include the colored dot.

Arrow color

Default arrows: var(--teal). 2px stroke for primary flow, 1px for secondary references. Return paths in a sequence diagram use arrow-muted (var(--rule-strong)) so direction is legible at a glance.

Caption

Italic Fraunces, var(--ink-faint), 14px, centered under the diagram with margin-top: 12px. See the captions on the figures above.

Exemplar reports

Read at least one before writing a fresh report. Exemplars are evidence of the language in real content — not templates to clone. This file is the canonical reference for vocabulary; exemplars show vocabulary deployed.

  • ~/notes/04_Research/atlas-on-alibaba-cloud-tour.html — dense ramp-up tour with the full component set. Reference for "everything this language can do at full scale."
  • ~/notes/04_Research/admin-backup-snapshots-overview.html — shorter operator reference (~70 lines minified). Same language at smaller scale; useful when the report is mid-length.

The two exemplars are visibly different in composition but instantly recognizable as the same language. That's the goal: locked brand stack, free composition.

If two reports in this language feel xeroxed, one of them is wrong. If two reports in this language feel like different brands, both of them are wrong.
