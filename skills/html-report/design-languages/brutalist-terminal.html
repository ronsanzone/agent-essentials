Design Language

Brutalist Terminal, monolithic mono

One typeface. One color for links. One color for the things that matter. The medium is the message, and the medium here is a green CRT terminal.

Personality

Brutalist Terminal is dark-mode brutalism. The whole document is set in one typeface — IBM Plex Mono — at multiple sizes and weights. Hierarchy is created by size and weight only; no serif, no display face, no exceptions. The background is near-black; the text is off-white; the links are terminal green. Headings prefix themselves with # and ## the way a markdown source file would, because the conceit is that you are reading the source.

A document set entirely in one mono is a document that admits it is text. That admission is the design.

Two accents and one alarm. Green (#00FF00) is the link color, the eyebrow dot, h1 italic-em, primary diagram strokes — the native old-terminal hyperlink. Amber (#FFB000) is for marginalia, h3 prefixes, pills, callout labels — the old-CRT secondary attention color. Red (#FF0000) is reserved for the warn / failure path and appears nowhere else.

[02]

When to pick this

Pick this when

Infrastructure documentation. CLI tool docs. SRE postmortems. Anything where the reader is most-comfortable in a terminal and the document is, in spirit, a long man page.

Don't pick this when

Long-form narrative reading (mono fatigues over 20+ minutes). Audiences expecting print-style typography. Reports where the prose carries more weight than the code blocks.

Voice rules

  • Headings prefix with > (h1), ## (h2), ### (h3) — characters are part of the heading.
  • Pull quotes wrap in """ (literal triple-quote) as a CSS pseudo-element.
  • Marginalia counters render as [01] [02] — inline brackets, no leading zero ambiguity.
  • Bullet lists use - as the marker, rendered via CSS pseudo-element in green.
  • Bold is allowed; italic is allowed but rare. Color is the primary accent device.

Warning: Do not introduce blue, magenta, or cyan. The terminal palette is green + amber + red on dark. Adding a fourth hue breaks the period authenticity.
[05]

Anti-patterns

  • No second typeface. Adding a sans or serif breaks the monolith.
  • No border-radius. Square corners only.
  • No box-shadow. Dark mode flat. The terminal had no shadows.
  • No gradients. Solid fills only.
  • No light mode. This language is dark-only.
  • No emoji. If a glyph isn't on a 1980s VT100, it doesn't belong.
  • No blue links. Links are #00FF00. Always.
  • No syntax themes beyond green/amber/red on dark. The four token classes stay within the palette.

Diagram philosophy

Diagrams are pure ASCII — rendered as monospace text inside SVG <text> blocks, not <rect> shapes. Box vocabulary is the strictest possible: + - | * < > only. No Unicode box-drawing, no curves, no anti-aliased shapes. Role shows through ink color and box-glyph choice: light boxes use +--+, hot boxes use +==+, alert frames bracket with !!!. The aesthetic is "this could be the output of cat diagram.txt."

Role-to-palette map

RoleGlyphsInkNotes
default+--+ | +--+#F0F0F0Light ASCII box, regular weight
hot+==+ | +==+#FFB000 boldHeavy = sides + amber ink — draws the eye

future+- - -+#555555Spaced dashes, ghosted

SVG typography

Each diagram is a single <text> with xml:space="preserve" and stacked <tspan x="10" dy="1.3em"> lines. Color and weight come from nested <tspan class="..."> spans. No <rect>, no <path>, no <marker> — the page renders a literal text file.

.ascii      { font: 14px "IBM Plex Mono", monospace; fill: #F0F0F0; white-space: pre; }
.ascii .hot  { fill: #FFB000; font-weight: 700; }
.ascii .ext  { fill: #888888; }
.ascii .step { fill: #FFB000; font-weight: 600; letter-spacing: 0.04em; }
.ascii .red  { fill: #FF0000; font-weight: 700; letter-spacing: 0.08em; }
.ascii .note { fill: #888888; }
LockedOnly standard ASCII glyphs: + - = | * < > ! /. Boxes are +--+ (default) or +==+ (hot). Step labels are [NN] in amber. Alert frame is !!! ALERT !!! in red.
FreeWidth of boxes, lifeline column positions, whether step numbers appear on simple diagrams.

Alert region pattern

Failure regions wrap in a literal !!!==…==!!! top-and-bottom rule, with red ! columns down each side. The inline alert uses a leading * glyph (the only star you'll see) and a RETRY POLICY ENGAGED red caps label, followed by a dim grey detail. No fills, no dashed borders, no SVG primitives — the frame is text. Visible in fig 02 between steps 03 and 04.

LockedFrame uses !!!=====...=====!!! for the rules and a single ! column on each side. Inline alert opens with * followed by a red caps snake-cased phrase. All red glyphs are bold #FF0000.
FreeFrame width, label copy, presence of an inner backoff … annotation.

Arrow vocabulary

Default: --> (single-line) and <-- for returns. Heavy / hot: ==> and <==. Async / fan-out: - - -> (spaced dashes). Warn arrows only appear inside !!! ALERT !!! frames and inherit the frame's red ink.

Caption

Mono lowercase dim text, 12px, left-aligned, prefixed by an amber // CSS pseudo. Captions are sentence-fragments; period optional. The caption describes what's in the ASCII — the ASCII is the diagram.
