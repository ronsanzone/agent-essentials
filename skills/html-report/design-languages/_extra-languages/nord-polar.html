Design Language

Nord Polar, arctic minimalism in long form

A cool, restrained register for developer documentation, changelogs, and open-source project sites — Scandinavian code-editor calm transposed to a technical document.

Status: candidate Mode: dark v1
LockedInter 300 h1 with bright-teal italic <em>, Inter 300 lede, mono caps eyebrow with --nord9 slate-blue dot, nord1 → nord0 linear gradient.
FreePadding, lede max-width, hero footer as .meta-row key/values or .meta pill chips.
01

Personality

Nord Polar feels like a Scandinavian code editor opened in a long-form document. The Frost layer — four shades of desaturated teal-to-slate-blue — creates an immediately recognizable cool coherence. Nothing is warm. Inter sans for prose and headings gives it the geometric calm of clean developer documentation; JetBrains Mono carries code, eyebrows, marginalia, and pills.

Bright teal --nord8 is the lead accent — it earns links, the italic-em in titles, the active TOC state, and primary diagram nodes. Slate-blue --nord9 is secondary — eyebrow dot, marginalia counter, ticket pills. Aurora red --nord11 is reserved for warnings, and only that.

Two reports in this language should feel like two pages from the same arctic editor — visibly the same voice, visibly not xeroxed.

The restraint is load-bearing. Nord is the variation that proves a technical document can carry its weight on cool color alone. Snow Storm light text on Polar Night surfaces; Frost accents that never shout; Aurora colors held in reserve. Swap the Frost lead for an Aurora color and the language becomes nord-aurora, a different document entirely.

02

When to pick this

Pick this when

Developer-facing documentation, CLI reference, SDK guides, open-source project sites. Changelogs with a single status type. Operator runbooks for an engineering audience that lives in a dark terminal. Anywhere a Scandinavian code-editor aesthetic feels native.

Don't pick this when

The audience expects a print-paper register (use editorial-parchment). The report needs to signal many semantic categories at once with color (use nord-aurora). The reader is a non-engineer or a public audience uncomfortable with dark themes (consider nord-snow).

Voice rules

  • Italic phrases inside h1 use <em> and are colored --nord8 bright teal.
  • Body italics are permitted; never bold body prose for emphasis — use italic.
  • Mono caps with 0.12–0.18em letter-spacing for eyebrows and meta keys.
  • Pull quotes are large (~24px), Inter light, bright teal, set off by a thin left rule.
  • The eyebrow dot is always --nord9 slate blue — the language's smallest secondary touch.

05

Anti-patterns

  • No warm tints anywhere — no cream, no parchment, no warm grey. Nord has zero warmth.
  • No system-font defaults (no Arial, no Roboto, no -apple-system fallback at the head of the stack).
  • No light-mode variation of this file — that is nord-snow, a different document.
  • No simultaneous use of all five Aurora colors as semantic signals — that is nord-aurora.
  • No bright/saturated accents outside the Nord palette (no electric magenta, no neon green).
  • No shadows tinted toward warm hues. Drop-shadows are flat near-black: rgba(20, 25, 35, 0.22).
  • No emoji in headings or callouts.
  • No decorative gradients beyond the very subtle Frost radial wash on body.
  • No serif display fonts — the language is sans-only.

07

Diagram philosophy

Diagrams here are quiet and precise — never decorative. Boxes have thin --nord2 borders, --nord1 fill, bright-teal --nord8 arrows. Text inside diagrams uses Inter (matching body) so diagram and prose remain in conversation. Aurora colors stay out of the diagram surface; only --warn (--nord11) is permitted, and only on the failure path.

Alert region pattern

When a diagram has a failure path, wrap the affected messages in an Aurora-red-tinted dashed rectangle and place an inline alert pill (nord11 @ 0.16α fill + --nord11 border + --nord11 mono caps text). The pattern is visible in the sequence diagram above between messages 03 and 04.

LockedWarn-tinted background at rgba(191, 97, 106, 0.06) + dashed --nord11 border. Inline alert pill stacks a colored dot, a mono-caps label, and an Inter italic detail.
FreeRegion size, exact alert pill copy, whether to include the colored dot.

Arrow color

Default arrows: --nord8. 2px stroke for primary flow, 1.5px for muted/return. Failure-path arrows use --nord11; async dispatch is dashed teal.

Caption

Inter light italic, --ink-faint, 13px, centered under the diagram with margin-top: 12px.

08

Exemplar reports

Read at least one before writing a fresh report. Exemplars are evidence of the language deployed in real content — not templates to clone.

  • nord-polar/cli-reference-tour.html — full CLI reference walkthrough; the language at full scale.
  • nord-polar/sdk-changelog-v3.html — release changelog at smaller scale; useful when the report is mid-length and the warn variant earns its keep.

The two exemplars are visibly different in composition but instantly recognizable as the same language. That's the goal: locked brand stack, free composition.

If two Nord Polar reports feel xeroxed, one is wrong. If two feel like different brands, both are wrong.
