Design Language

Flexoki Prose, one typeface, one accent

A writer's-tool register. Hierarchy through size and weight alone — no font switch, no second color. For reports that should feel like notebook pages, not magazine spreads.

Status: candidate Audience: engineer-internal Single accent
LockedSingle typeface across body, headings, and code. Orange italic <em> in titles. Orange-only accent — no secondary color. No .num marginalia counter.
FreePadding, lede max-width, hero footer composition, exact heading sizes within the 32/21/15 scale.
01

Personality

Flexoki Prose is the monastic variant. One typeface — iA Writer Quattro V (a "duospaced" sans that reads like prose but has a mono-like rhythm) — used for body, headings, and code. Since iA Writer Quattro V isn't available on Google Fonts, the web fallback is IBM Plex Sans used uniformly. Either way, the discipline is the same: hierarchy comes from size and weight alone. No second typeface, no register switch.

The accent is orange. Just orange. Used on links, the italic-em phrase in the hero title, the active TOC entry, the callout left-border, the pull-quote color, the details summary, the pill border, the progress bar, and the eyebrow dot. There is no secondary color. There is no separate warn color. A "warn" callout in this language renders identically to a default callout; the warning is communicated by the <strong>Warning:</strong> label and the prose, not by a palette shift.

When the only tool is typography, the writer has to actually say what they mean.

This is a constrained language by design. If a report needs a dedicated warn palette, a hot/cold accent pair, or marginalia numbering, pick a different variant. Flexoki Prose's value is the constraint.

02

When to pick this

Pick this when

The report is short, focused, and writer-driven. The author wants the document to feel like notes from a sustained thinking session — not a magazine layout. The content's hierarchy is genuinely flat (no multi-level taxonomy of statuses, severities, or categories that would benefit from color encoding).

Don't pick this when

The report has multiple status classes (warn/success/info) that need distinct visual treatment. The audience is skimming and needs color cues to navigate. The content is operationally critical and the warn callout must visually scream — Flexoki Prose's monochrome warn is intentionally quiet.

03

Locked brand stack

The intended typeface is iA Writer Quattro V — a duospaced sans designed for writing tools. Because iA Writer Quattro V is not available on Google Fonts, the realistic web compromise is to load IBM Plex Sans and use it uniformly. Document this honestly in your reports.

<link href="https://fonts.googleapis.com/css2?family=IBM+Plex+Sans:wght@300;400;500;600;700&display=swap" rel="stylesheet">

Font stack variables

/* All three roles resolve to the same family. The aliases exist so canonical
   components written for the editorial register still resolve. */
--serif: "iA Writer Quattro V", "IBM Plex Sans", -apple-system, sans-serif;
--sans:  "iA Writer Quattro V", "IBM Plex Sans", -apple-system, sans-serif;
--mono:  "iA Writer Quattro V", "IBM Plex Sans", -apple-system, sans-serif;

If iA Writer Quattro V is installed locally (it ships with the iA Writer app and is also available from iA's GitHub at github.com/iaolo/iA-Fonts), it will be used; otherwise the rendering falls back to IBM Plex Sans across the board. The brand survives the fallback because the hierarchy doesn't depend on which sans loads.

Hierarchy rules

  • h1 — 40–64px (clamp), weight 700.
  • h2 — 32px, weight 700.
  • h3 — 21px, weight 600.
  • h4 — 15px, weight 600.
  • body — 16px, weight 400, line-height 1.7.
  • Letter-spaced caps (0.12–0.18em tracking) signal mono-register elements: eyebrows, table headers, meta keys, contract labels.

Voice rules

  • Italic for ledes, pull quotes, and italic-em in titles.
  • Never bold body prose for emphasis — use italic instead. Bold is reserved for headings and the <strong>Label:</strong> prefix on callouts.
  • The eyebrow dot is always var(--orange) — the only accent in the language.
  • Links are orange; hover collapses to ink. No purple visited state.

05

Anti-patterns

  • No second typeface. The whole point is one family.
  • No second accent color. Orange-only.
  • No marginalia counter. The .num element stays in markup for structural compatibility but is display: none.
  • No distinct warn palette. A warn callout is a default callout with a warning word, not a warning color.
  • No bold body prose for emphasis — italic only.
  • No cool-grey neutrals; the ramp stays warm.
  • No system-font defaults. IBM Plex Sans must load.
  • No saturated hue outside orange. No green checkmarks, no red errors, no blue links.
  • No emoji.
  • No decorative gradients beyond the very-faint paper grain.

07

Diagram philosophy

Diagrams in Flexoki Prose are even quieter than in Paper or Ink. Every box is a thin-bordered rectangle. Arrows are orange — the only color in the diagram. Nodes that need to stand out use a soft orange wash (the hot role); nodes that recede use the paper-card default. external, store, and warn all collapse visually into "either default or hot" — the language doesn't carry enough color budget to distinguish them, and that's fine, because Flexoki Prose is meant for reports where those distinctions don't carry weight.

Alert region pattern

Wrap failure-path messages in an orange-tinted dashed rectangle and place an inline alert pill (#F7E2D0 fill + orange border + orange caps text). The pattern is visible in the sequence diagram between messages 03 and 04. Note that in Flexoki Prose this region looks identical to a hot region — that's intentional. The reader sees "something orange is happening" and reads the prose to find out which.

LockedRegion background uses rgba(188, 82, 21, 0.05) + dashed orange border. The inline alert pill stacks an orange dot, orange caps label, and same-family italic detail.
FreeRegion size, exact alert pill copy, whether to include the colored dot.

Arrow color

All arrows are var(--orange) at 2px. Return paths use arrow-muted (var(--rule-strong)) — the only non-orange stroke in the diagram vocabulary, and only used to distinguish direction. Async paths add stroke-dasharray: 6 3.

Caption

Same-family italic, var(--faint), 14px, centered with margin-top: 12px.

Exemplar reports

Read at least one before writing a fresh report in this language. Flexoki Prose is the variant most likely to be used for short, opinion-driven docs — design memos, architectural decision records, retrospectives — so the exemplars skew shorter than Paper's or Ink's.
