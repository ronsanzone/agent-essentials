Design Language / Chrome

Synthwave Chrome, dialed back

The synthwave register restrained enough for a 20-minute technical read — for tech magazines, long product analyses, and any document where the typography should set a mood without becoming the subject.

Status: shipped Audience: engineer-internal v1

01

Personality

Chrome is synthwave dialed back to a working register. Same deep-navy #0c0c1e base, same cyan/magenta signal pair — but used measuredly. Cyan is the lead, but it appears as link color, italic-em, primary diagram arrows, and pull-quote ink, not as a glow on every element. Magenta is reserved: marginalia number, eyebrow dot, contract label, h1 underline. The neon never leaves the palette, but it never overwhelms either.

Synthwave can be a register for a real 20-minute read — if you trust the reader and stop trying to prove the aesthetic on every line.

Display is Orbitron, used only for h1. Section h2 and below are Inter — a working sans that disappears into the text. JetBrains Mono carries code, eyebrows, table headers, and marginalia. The serif/display split is intentional: Chrome wants the reader to feel the synthwave mood at the top of the page, then to read.

02

When to pick this

Pick this when

Long-form technical reads where the audience and content are adjacent to retro/synth themes. Product analyses for retro-themed hardware. Editorial issues of a synthwave-themed tech magazine. Any 15–25 minute read where Outrun would distract but Editorial Parchment would feel out of place.

Don't pick this when

The audience is not retro-adjacent — Chrome is still a themed register, just a quiet one. The report is short editorial (Outrun is better). The document needs a warm/serious register without theme (Editorial Parchment).

Voice rules

  • Orbitron is the one display moment. h1 only — never h2 or h3.
  • The h1 <em> is cyan, weight 400 (lighter than the surrounding 500).
  • Pull quotes are Inter italic 24px in cyan, with a 1px cyan left rule. No text-shadow glow.
  • The eyebrow dot is magenta. It is the one magenta accent visible above the fold.
  • Italic for emphasis in body. Bold also acceptable; don't combine the two.
  • No text-shadow anywhere in body. Chrome is a flat-color register.

05

Anti-patterns

  • No text-shadow glow on text. Anywhere.
  • No sunset gradient hero. The hero is a flat deep-navy with a 1px cyan bottom rule.
  • No Orbitron at h2 or below — at small sizes it reads as costume.
  • No box-shadow glow on cards. Cards have a flat rule-strong border.
  • No Audiowide, VT323, or other display synthwave faces. Chrome stays with Orbitron + Inter + JetBrains Mono.
  • No magenta in body prose. Magenta is reserved for the eyebrow dot, marginalia number, contract label, and SVG external role.
  • No light mode. Chrome is dark-only.
  • No emoji in section headings.

Diagram philosophy

Diagrams in Chrome are precise and quiet. 1.5px borders, deep-navy fills, cyan arrows. Inter for diagram text (matching body). hot nodes get a cyan border; external nodes get magenta — the only place magenta appears in a diagram body.

Alert region pattern

Wrap failure-path messages in an orange-sun-tinted dashed rectangle, place an inline alert pill (orange-sun-tinted fill, orange-sun border, orange-sun mono caps text). Visible in the sequence above between messages 03 and 04.

LockedBackground rgba(253, 138, 38, 0.05) + dashed orange-sun border. Inline pill: orange-sun dot + orange-sun mono caps label + ink-faint italic detail.
FreeRegion size, alert copy, presence of the dot.

Arrow color

Default arrows are cyan at 2px. Return paths use arrow-muted (rule-strong navy). Async uses dashed cyan (stroke-dasharray: 6 3). No magenta arrows — magenta is reserved for step numbers and external borders.

Caption

Inter italic, ink-faint, 13px, centered under the diagram with margin-top: 12px.

Exemplar reports

Read at least one before writing a Chrome report.

  • ~/notes/04_Research/synth-engine-architecture-tour.html — long-form architectural tour of a software synthesizer. Reference for Chrome at full 20-minute scale.
  • ~/notes/04_Research/midi-spec-revision-analysis.html — technical analysis of a spec change. Chrome at medium length; useful when content is dense but theme is retro-adjacent.

Two reports should feel like two long features in the same themed magazine: visibly the same publication, visibly not the same article.

Restraint is what makes Chrome a real reading register. Add one glow and the language collapses into a worse Outrun.
