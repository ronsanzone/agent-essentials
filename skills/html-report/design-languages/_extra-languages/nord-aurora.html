01

Personality

Nord Aurora is the variation that puts Nord's full palette to work. Polar Night still supplies surfaces and Snow Storm still supplies ink, but the Frost layer is demoted: Frost teal becomes the cool secondary signal on links and a colder counterweight, while Aurora orange --nord12 steps into the lead-accent slot. Each Aurora color earns a distinct semantic role — green is success, red is danger, yellow is caution, orange is highlight, purple is special. The result is a document that can carry many status types at once without leaving the Nord palette.

This is the variation for a report that needs to signal. A multi-component changelog where some changes are breaking, some are deprecations, some are new features, and some are housekeeping — that report wants Nord Aurora. A simpler narrative wants nord-polar.

Nord Aurora is what proves Nord is more than four shades of cool — it's a complete semantic palette when you actually need one.

The body and mono fonts match nord-polar exactly (Inter + JetBrains Mono). The two dark variants are siblings — the typography is identical; only the color logic differs.

02

When to pick this

Pick this when

Changelogs with breaking / deprecated / new / fixed categories. Feature matrices where status columns need color signaling. Release-readiness reports with go / no-go / caveats. Anywhere five or more distinct semantic categories must be legible at a glance.

Don't pick this when

The narrative has one tone (use nord-polar). The audience reads on light surfaces (use nord-snow). The report is short and decorative color would feel busy. The audience expects a print-paper register (use editorial-parchment).

Voice rules

  • Italic phrases inside h1 use <em> and are colored Aurora orange --nord12 — the lead accent. This is the visual hook that separates Aurora from Polar.
  • Body italics are permitted; never bold body prose for emphasis — use italic.
  • Mono caps with 0.12–0.18em letter-spacing for eyebrows and meta keys.
  • Pull quotes are large (~24px), Inter light, Aurora orange, set off by a thin purple left rule.
  • The eyebrow dot is always --nord15 purple — Aurora's secondary touch.
  • Links use Frost teal --nord8 — Frost is the colder secondary in this variation, deliberately. This is what makes Aurora and Polar different: in Polar, teal leads and Aurora is reserved; here, the reverse.

05

Anti-patterns

Aurora is the variation that permits more color than its siblings. The anti-patterns are about discipline within that permission, not about staying monochrome.

  • No semantic colors from outside the Aurora layer. If you need a sixth meaning, you have too many; collapse two into one.
  • No promotion of --nord8 bright teal to the lead-accent slot — that is nord-polar. Here, Frost is demoted on purpose.
  • No warm tints outside the documented Aurora orange — no terracotta, no rust, no amber. Aurora orange is the only orange.
  • No mixing of Aurora orange and Aurora red in a single callout — the pair reads as confused signaling.
  • No light-mode variation of this file — that is a future variation.
  • No system-font defaults at the head of the stack.
  • No emoji-as-status-signal. Color is the status signal; emoji is decoration.
  • No heavy gradients beyond the subtle Aurora-tinted radial washes on the hero.

4. Callout family .callout

Aurora supports the full callout family — five variants, each mapped to one Aurora color. This is the variation's signature; the others restrict the family to default + warn.

Note: Default callouts use Aurora-orange wash + 4px --nord12 left border.
Done: Success callouts use Aurora-green wash + 4px --nord14 left border. Use for "this has shipped", "this passed", "this completed".
Warning: Warn callouts use Aurora-red wash + 4px --nord11 left border. Use for deprecation, failure paths, danger.
Caution: Caution callouts use Aurora-yellow wash + 4px --nord13 left border. Use for "verify before relying on this", "caveat applies", "edge case".
Featured: Special callouts use Aurora-purple wash + 4px --nord15 left border. Use sparingly — featured releases, highlights, callouts you want to read different from everything else.
Locked4px Aurora-color left-border + matching wash background. Geometry is identical across variants; only the color changes. Each variant maps 1:1 to one Aurora color.
FreeBorder-radius, padding, presence of icon, exact wash opacity (0.12–0.20).

Exemplar reports

Read at least one before writing a fresh report. Exemplars are evidence of the language deployed in real content — not templates to clone.

  • nord-aurora/sdk-v3-release-notes.html — major release with breaking changes, deprecations, new APIs, and featured upgrades. The language at full Aurora-mapping scale.
  • nord-aurora/quarterly-feature-matrix.html — feature matrix with multi-status color signaling across providers. Useful when the report's main job is comparing categories.

The two exemplars are visibly different in composition but instantly recognizable as the same language. That's the goal: locked brand stack, free composition, full Aurora mapping.
