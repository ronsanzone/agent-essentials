Design Language

Anthropic Clay, the product warmth

The product-UI register — sans-only Inter on warm cream, terracotta lead everywhere. The language a Claude product page would feel like if it had to render a long report.

Status: candidate Audience: product-internal v0.1
LockedInter throughout (no serif), <em> as upright terracotta accent (not italic), warm cream #F9F7F2 background with radial terracotta glow on hero.
FreePadding, lede max-width, hero footer as .meta-row or .meta.
01

Personality

Anthropic Clay is the product-UI register. It feels like a long-form page lifted from Claude's own product surfaces: warm cream backgrounds, generous terracotta presence, soft-cornered cards with a quiet inset highlight, no serif in sight. Where anthropic-paper is the company speaking, Clay is the product speaking — operator docs, internal release notes, runbooks for a feature team, anything that should feel native to the Claude UI it sits next to.

If Anthropic Paper is a research bulletin, Clay is the surface around the chat box — warmer, more handled, more terracotta.

Lead accent is --terracotta: links, eyebrow text, active TOC pill, primary diagram nodes, callout borders, code keywords. Secondary is --terracotta-deep for hover/active deepening — a shade not a separate hue. Warn is --amber, the only non-terracotta accent in the language and reserved for the warn callout and warn diagram nodes.

No serif. The absence of Lora is intentional: this is product UI, not editorial. Italic emphasis becomes upright terracotta instead — a treatment that reads as "highlighted in the UI" rather than "italicized in a paragraph."

02

When to pick this

Pick this when

The report lives adjacent to product UI — feature runbooks, operator references for Claude product features, internal release notes that link back to product surfaces, design-review documents for in-product changes. Short to medium length (5–15 minutes reading time) where the cream warmth and soft-card chrome don't fatigue the reader.

Don't pick this when

The report is for external publication — pick anthropic-paper (corporate brand canonical). The report is an AI-safety preprint or academic write-up — pick anthropic-research. The report is a 20+ minute engineering ramp-up with citation density — editorial-parchment handles that better.

03

Locked brand stack

Two faces only: Inter for everything, JetBrains Mono for code and labels. No serif. The absence of a third font is part of the language's identity — it keeps the document feeling like a product page rather than a publication.

Font role assignments

RoleFontNotes
Body prose, headings, ledes, summariesvar(--sans) — Inter15.5px body; weight 500–600 for display; ss01 + cv11 features on for a/g forms

Voice rules

  • Inter everywhere except code. No serif emphasis — <em> becomes upright terracotta, not italic.
  • Inter feature settings ss01 (alt a) and cv11 (single-storey g) lend a softer product feel; both are part of the locked brand stack.
  • Mono caps with letter-spacing for eyebrows, meta-row labels, marginalia counter, pill text.
  • Pull quotes are sans-serif weight 500 in --terracotta-deep, set off by a 3px terracotta left rule — not a hairline.
  • The eyebrow dot is always --terracotta at 7px (one pixel larger than Paper's 6px) — the language wears its lead color confidently.

05

Anti-patterns

  • No serif fonts. No Lora, no Source Serif, no Georgia fallback as a body face. Inter is the body face.
  • No italic emphasis. <em> renders upright in --terracotta-deep; the geometry "italic body text" does not exist in this language.
  • No blue, green, purple, or pink anywhere. Terracotta + amber + cream is the complete palette.
  • No dark themes — cream is load-bearing.
  • No saturated/web-1.0 terracotta (#cc4400-style). Clay's terracotta is muted — read OKLCH lightness ≈ 0.70, chroma ≈ 0.14.
  • No shadows heavier than 0 8px 22px rgba(80, 50, 30, 0.045). The card chrome is soft, not a Material elevation.
  • No emoji in headings, callouts, or pills.
  • No icon library imports — icons, if any, are inline SVG with terracotta stroke.

3. Card .card

A self-contained card

Cards in Clay carry a soft inset highlight at the top edge — a subtle white inset shadow that catches the cream and reads as "this is a UI surface, not a slab of paper." Border-radius is tighter (14px) than Paper's 18px.

Lockedbg-card fill, 1px solid var(--rule), inset top highlight 0 1px 0 rgba(255,255,255,0.6) inset, drop shadow 0 8px 22px rgba(80,50,30,0.045).
FreeBorder-radius (12–16px), padding.

4. Callout .callout

Note: Default callouts use --terracotta-soft background with a full --terracotta border and a 4px left edge. The full border (not just a left bar) is Clay's product-UI tell.
Warning: Warn variant uses amber — the only non-terracotta accent. Reserve for deprecation, failure paths, and known gaps.
Done: Success uses --terracotta-deep on a deeper terracotta wash. Clay does not have a green success — the depth of terracotta carries the signal.
LockedFull 1px border + 4px left edge. Default = terracotta family; warn = amber; success = terracotta-deep family. No green.
FreeBorder-radius, padding, presence of bold label.

5. Pull quote .quote

The stickiest sentence in a section — the one a skimming reader must absorb. One per section maximum.
LockedInter weight 500 (not italic) in --terracotta-deep, 22px, 3px terracotta left rule (not hairline).
FreeExact font-size (20–24px), padding.

6. Details block details

Step 1. Initial step (open by default)

Each details is a bordered, rounded card with a rotating chevron marker. The marker is terracotta; the summary text is ink (not terracotta) — Clay reserves the terracotta on summaries for the marker only.

Step 2. Subsequent step

The chevron rotates 90° on open. This is Clay's product-UI animation language — small, soft, terracotta.

LockedBordered rounded card per details, terracotta chevron marker that rotates on open, ink summary text (not terracotta).
FreeBorder-radius, padding, exact summary font-weight (500–600).

Diagram philosophy

Diagrams in Clay feel like Claude product diagrams — soft, warm, terracotta-led. Text inside diagrams uses Inter (matching body) so the diagram and the surrounding prose share a typographic register. The role map leans heavily on terracotta variants rather than introducing new hues.

08

Exemplar reports

Exemplars do not yet exist for this candidate language. Once Clay is promoted, the first reports to use it should be Claude product feature runbooks or internal release notes that link back to product surfaces.

  • Expected first use: a Claude feature operator runbook (e.g., "Memory ops reference" or "Skills runtime mechanics for on-call").
  • Expected second use: an internal release note that wants to feel like a product page.
If two reports in Clay don't feel like they belong adjacent to a Claude product surface, one of them has drifted into Paper territory.
